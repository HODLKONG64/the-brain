# READMENOWBOT — GK BRAIN Telegram Bot Guide

Quick guide to the GK BRAIN Telegram bot: what it does, how to configure it, and how to troubleshoot it.

---

## What the Bot Does

The GK BRAIN Telegram bot (powered by `gk-brain.py`) runs every 2 hours and:

1. **Generates two consecutive lore posts** in the voice of GK (UK graffiti artist / DJ / parkour runner / carp fisherman / web3 entrepreneur).
2. **Attaches an AI-generated image** to Post 2 (generated by Grok, streamed directly — never stored on disk).
3. **Pins Post 2 silently** in each target channel/group.
4. **Replies to user messages** that contain project-relevant keywords (rate-limited to 20 replies per user per 24 hours).

---

## Bot Setup Checklist

- [ ] Create a bot via [@BotFather](https://t.me/BotFather) → copy the token
- [ ] Set `TELEGRAM_BOT_TOKEN` secret in GitHub repo settings
- [ ] Set `CHANNEL_CHAT_IDS` to a comma-separated list of target chat IDs (e.g. `-1001234567890,-1009876543210`)
- [ ] Add the bot to each target channel/group as an **admin** with **Post Messages** + **Pin Messages** rights
- [ ] Optionally set `MESSAGE_THREAD_ID` if posting into a specific topic thread

---

## Getting Chat IDs

Add `@userinfobot` or `@RawDataBot` to a channel temporarily, or send a message and call:

```
GET https://api.telegram.org/bot<TOKEN>/getUpdates
```

The `chat.id` field in the response is what you need. Channels use negative IDs (e.g. `-100...`).

---

## Message Format

**Message 1** — Plain text lore post. Max 4,096 UTF-8 characters.
**Message 2** — Lore post + AI-generated image as caption. Max 1,024 UTF-8 characters.

A `time.sleep(2)` is called between Message 1 and Message 2 to comply with Telegram rate limits (DB-12).

---

## Rate Limits

See `TELEGRAM-BOT-API-RULES.md` for the full rule set. Key limits:
- 30 messages/second across all chats combined
- 1 message/second per individual chat
- On `429 Too Many Requests`: sleep `retry_after` seconds, then retry once

---

## Error Reference

| Error | Cause | Action |
|---|---|---|
| `401 Unauthorized` | Bot token invalid | Re-generate token via BotFather |
| `403 Forbidden` | Bot not admin in chat | Add bot as admin with correct rights |
| `400 Bad Request` | Malformed message (e.g. too long) | Check text length; fix template |
| `429 Too Many Requests` | Rate limit hit | Sleep `retry_after`; retry once |

---

## Reply Tracking

`bot-state.json` stores the last processed `update_id` to prevent replaying old messages.
`reply-tracker.json` stores per-user reply counts and dates to enforce the 20-reply/day cap.

Both files are committed back to `main` after every run.

---

## Disabling the Bot

To temporarily disable posting without stopping the workflow, remove all entries from `CHANNEL_CHAT_IDS` or leave it empty. The brain will log a warning and skip the posting loop.
