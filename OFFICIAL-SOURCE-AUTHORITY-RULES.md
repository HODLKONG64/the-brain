## Rule 2 — Canon Verification Required Before Any Name Use
Any proper noun (person, place, faction, item, event name) found on ANY website — including fan sites, social media, third-party blogs — must be verified against the official sources list before it can be used in lore, wiki updates, or Brain 2 data.

## Rule 3 — Non-Canon Name Handling
If a name is NOT found on any official source:
- It is classified as NON-CANON
- It must NOT appear in any lore post
- It must NOT be written to the Fandom wiki
- It must NOT be saved to brain1-canon.json or any Brain 2 data file
- It must be logged to non-canon-names.log

## Rule 4 — Canon Name Handling
If a name IS found on an official source:
- It is classified as CANON and may be used freely in lore, wiki updates, and Brain 2 data
- The confirming URL must be recorded alongside any use of the name

## Rule 5 — Override Rule (Highest Priority)
Any new official content published on an official source URL automatically supersedes all agent-generated content for that subject. Brain 3 must scrap its temporary version and switch to the new official data immediately on next cycle.

## Rule 6 — No Fan Lore Promotion
If crawling any non-official website reveals lore, characters, or world-building invented by fans or third parties, that content must be ignored entirely — it cannot influence Brain 2 lore, wiki content, or Brain 3 output.

## Rule 7 — Wiki Citation Health Check
All citation URLs on wiki pages must be verified as live (HTTP 2xx) before any wiki write. Dead links must be repaired with a replacement URL or removed. (Implemented by wiki-citation-checker.py)

## Rule 8 — Source Attribution
Every piece of real data injected into lore must have its source URL recorded in web-lore-output.json for auditability.

## Rule 9 — Wiki Pre-Write Citation Audit
Before any page is written to the Fandom wiki, wiki-citation-checker.py must run and verify all existing citations on that page. No wiki write proceeds if the citation checker errors — it must fail gracefully and log.

## Rule 10 — Telegram Lore Posts Are NOT Wiki Sources
Content generated by Brain 2 (gk-brain.py) for Telegram is for Telegram only. Brain 2 lore posts must NEVER be used as a source for wiki updates or Brain 3 real-data injection. Only content crawled from official source URLs (Rule 1) may feed the wiki or Brain 3.
