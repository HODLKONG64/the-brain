## ⚠️ TELEGRAM CONTENT IS NEVER A WIKI SOURCE

Content generated by the Telegram bot/agent (Brain 3 / gk-brain.py Telegram posts) must
**NEVER** be used as a source for wiki updates. Telegram posts are **output only**.

The wiki must only be populated from the official web sources listed below and in
`gkandcryptomoonboywebsitestosave.md`.

---

## ⚠️ wiki-citation-checker.py Is a Health Tool Only

`wiki-citation-checker.py` checks for **dead links on existing wiki pages**. It is a
maintenance/health tool. It must **NEVER** be used as a gate or filter that blocks new
wiki content from being written. Content sourced from the official source list below must
**NEVER** be blocked by citation-checking logic.

## Rule 2 — Canon Verification Required Before Any Name Use
Any proper noun (person, place, faction, item, event name) found on ANY website — including
fan sites, social media, third-party blogs — must be verified against the official sources
list before it can be used in lore, wiki updates, or Brain 2 data.

## Rule 3 — Non-Canon Name Handling
If a name is NOT found on any official source:
- It is classified as NON-CANON
- It must NOT appear in any lore post
- It must NOT be written to the Fandom wiki
- It must NOT be saved to brain1-canon.json or any Brain 2 data file
- It must be logged to non-canon-names.log

## Rule 4 — Canon Name Handling
If a name IS found on an official source:
- It is classified as CANON and may be used freely in lore, wiki updates, and Brain 2 data
- The confirming URL must be recorded alongside any use of the name

## Rule 5 — Override Rule (Highest Priority)
Any new official content published on an official source URL automatically supersedes all
agent-generated content for that subject. Brain 3 must scrap its temporary version and
switch to the new official data immediately on next cycle.

## Rule 6 — No Fan Lore Promotion
If crawling any non-official website reveals lore, characters, or world-building invented
by fans or third parties, that content must be ignored entirely — it cannot influence Brain 2
lore, wiki content, or Brain 3 output.

## Rule 7 — wiki-citation-checker.py Is a Dead-Link Health Tool ONLY
`wiki-citation-checker.py` is used exclusively to check for dead/broken links on existing
wiki pages. It must NOT act as a content gate. It must NOT prevent new content from being
added to the wiki. It must NOT reject content from the official sources listed in Rule 1.

## Rule 8 — Source Attribution
Every piece of real data injected into lore must have its source URL recorded in
web-lore-output.json for auditability.

## Rule 9 — Wiki Pre-Write Citation Health Check (Non-blocking)
Before any page is written to the Fandom wiki, wiki-citation-checker.py may run to check
existing dead links on that page. This check is advisory only — a dead link on an existing
page must NOT prevent new content from being appended. It must fail gracefully and log.

## Rule 10 — Telegram Lore Posts Are NOT Wiki Sources
Content generated by Brain 3 (gk-brain.py) for Telegram is for Telegram only. Brain 3
lore posts must NEVER be used as a source for wiki updates or Brain 1/2 data injection.
Only content crawled from official source URLs (Rule 1) may feed the wiki.

## Rule 11 — Wiki URL Canonical Form
The canonical wiki URL is: https://gkniftyheads.fandom.com/wiki/GKniftyHEADS_Wiki
The old Main_Page URL was moved/renamed by Fandom SEO in March 2026 — both URLs resolve
to the same page. Always use the URL stored in the agent's FANDOM_WIKI_URL environment
variable — do not hardcode the page path in agent code.
