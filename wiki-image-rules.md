# wiki-image-rules.md
# Image Standards for GK BRAIN Wiki Pages

These rules govern how the GK BRAIN agent **finds, cites, and embeds images** on
every Fandom wiki page it creates or updates.  All new and edited pages must
comply with these rules.

---

## 1. Canonical Image Sources

Images must be sourced **in priority order**:

| Priority | Source                                         | Notes                                      |
|----------|------------------------------------------------|--------------------------------------------|
| 1        | Official GraffPUNKS Substack posts             | Highest authority; always preferred        |
| 2        | Official GKniftyHEADS NFT collection art       | AtomicHub / NeftyBlocks canonical files    |
| 3        | Official GKniftyHEADS.com or GraffPUNKS.live   | Website hero images, press-kit art         |
| 4        | AI-generated images produced by GK BRAIN       | Acceptable when no official art exists     |
| 5        | Social media (X/Twitter, Instagram, Facebook)  | Only when no other source is available     |

**Never use:**
- Unverified fan art unless explicitly confirmed by the official accounts.
- Stock photography or unrelated images.
- Hotlinked external images that may disappear (always upload to wiki).

---

## 2. Upload Rules

1. **Always upload** the image to the wiki via `action=upload` before embedding it.
   Direct hotlinking to external CDNs is forbidden.
2. The upload filename must follow the pattern: `{Entity_Title}.{ext}`
   - Use underscores instead of spaces: `Bitcoin_Kid.jpg`
   - Allowed extensions: `.jpg`, `.jpeg`, `.png`, `.gif`, `.svg`, `.webp`
   - Use `.jpg` as the default when extension is unknown.
3. Set the upload **comment/summary** to:
   `"Image for [[Page Title]] — uploaded by GK BRAIN"`
4. Pass `ignorewarnings=true` to the MediaWiki upload API to avoid stale-cache
   rejections on re-uploads.

---

## 3. Embedding Rules

### 3.1 Standard embed (single image per page)
```mediawiki
[[File:Entity_Title.jpg|thumb|right|Caption text here.]]
```
- Always use `thumb` and `right` for the main page image.
- The image block goes **immediately after the infobox** (or at the top of the
  lead section when no infobox exists).

### 3.2 Gallery (multiple images)
When a page needs more than one image (e.g. an NFT collection showing several
variants):
```mediawiki
<gallery>
File:Entity_Title_1.jpg|Caption 1
File:Entity_Title_2.jpg|Caption 2
</gallery>
```
- Place galleries at the **end of the relevant section body**, not at the top.

### 3.3 Caption rules
- Captions must be factual and concise (max 15 words).
- Do **not** embed wikilinks inside captions.
- Captions must credit the source where known, e.g.
  `"Bitcoin Kid. Art © GKniftyHEADS."`

---

## 4. Alt-text and Accessibility
- The `thumb` keyword in `[[File:…|thumb|…]]` makes MediaWiki auto-generate
  a title attribute from the caption.  No additional `alt=` parameter is needed
  for Fandom wiki pages.

---

## 5. Image Citations in References
Whenever an image is sourced from an official URL, cite that URL in the
`== References ==` section using a `<ref>` tag on the caption line:

```mediawiki
[[File:Bitcoin_Kid.jpg|thumb|right|Bitcoin Kid.<ref>{{cite web
| url    = https://graffpunks.substack.com/p/bitcoin-kid
| title  = Bitcoin Kid — GraffPUNKS Substack
| access-date = 2026-03-18
}}</ref>]]
```

For AI-generated images, the citation should read:
```mediawiki
<ref>Image generated by GK BRAIN AI agent based on the Character Bible.</ref>
```

---

## 6. Image Freshness
- When new official art is published on an authoritative source (see §1), the
  existing wiki image **must be replaced** within the next 2-hour update cycle.
- On replacement, increment the upload filename or use `ignorewarnings=true`
  to overwrite the existing file revision.
- Update the caption to reflect any new factual details.

---

## 7. Missing Images
If no compliant image can be found:
1. Leave the `image_filename` field blank in the infobox (`| image =`).
2. Do **not** embed a placeholder or broken `[[File:…]]` tag.
3. Add the page to `[[Category:Pages needing images]]` so it can be tracked.

---

## 8. Filename Collision Avoidance
Before uploading, `wiki-page-builder.py` derives the filename from the entity
title.  If two entities share a title fragment, append the entity type:
```
Bitcoin_Kid_character.jpg   (not just Bitcoin_Kid.jpg)
```
The `_derive_image_filename()` helper in `wiki-page-builder.py` handles this
automatically.

---

## 9. Compliance Checklist

Before any wiki page is published by GK BRAIN, verify:

- [ ] Image sourced from the highest-priority available source.
- [ ] Image uploaded to wiki (not hotlinked externally).
- [ ] Filename matches `Entity_Title.ext` pattern (underscores, safe chars).
- [ ] Embedded with `[[File:…|thumb|right|Caption.]]` directly after infobox.
- [ ] Caption is ≤ 15 words, factual, and credits the source.
- [ ] Citation `<ref>` added in `== References ==` when source URL is known.
- [ ] Page added to `[[Category:Pages needing images]]` if no image is available.
