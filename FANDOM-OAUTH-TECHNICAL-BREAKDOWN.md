## OAuth 1.0a Flow
OAuth 1.0a is a protocol that allows clients to access resources on behalf of a user. The flow typically includes:
1. **Request Token**: The client requests a temporary token from the service provider.
2. **User Authorization**: The user is redirected to a service provider page to authorize the application.
3. **Access Token**: After authorization, the client exchanges the request token for an access token, which can be used to access protected resources.

```plaintext
Flow Steps:
1. Obtain Request Token
2. Redirect User for Authorization
3. User Grants Access
4. Obtain Access Token
5. Access Protected Resources
```

## Redirect=no Parameter Meaning
The `redirect=no` parameter is often used to indicate that the authorization flow should not include a redirection response to the user's browser. Instead, it may return a token or an error directly in the response. This can be useful for embedded applications or server-to-server communication, where you do not want to redirect the user.

## CSRF Token Handling
Cross-Site Request Forgery (CSRF) tokens are used to protect against malicious exploits:
- A CSRF token is generated by the server and associated with the user's session.
- Upon each request that modifies state (like a POST request), the CSRF token must be included and validated by the server to ensure the request is genuine and made by the user.

### Example CSRF Handling:
```javascript
// When sending a request:
const csrfToken = document.querySelector('meta[name="csrf-token"]').getAttribute('content');
fetch('/api/resource', {
    method: 'POST',
    headers: {
        'X-CSRF-Token': csrfToken,
        'Content-Type': 'application/json'
    },
    body: JSON.stringify(data)
});
```

## Rate Limiting
Rate limiting controls how often a user can make requests to an API within a specified time period. This is important to prevent abuse and ensure fair usage of resources. Fandom APIs may implement rate limiting typically measured in requests per minute or per hour.

### Example Code for Handling Rate Limit:
```javascript
async function fetchWithRateLimit(url) {
    const response = await fetch(url);
    if (response.status === 429) { // Too many requests
        const retryAfter = response.headers.get('Retry-After');
        setTimeout(() => fetchWithRateLimit(url), retryAfter * 1000);
    }
    return response.json();
}
```
